纯前端 WASM 翻译引擎

让网页翻译变得简单高效

翻译大师是一个基于 Transformers.js 的纯前端翻译库。 它使用 ONNX Runtime 在浏览器中运行机器翻译模型， 无需任何服务器依赖，保护用户隐私，支持离线使用。 无论是简单的文本翻译还是复杂的 DOM 自动翻译， 翻译大师都能轻松应对。

🚀

智能模型路由

根据语言对自动选择最优模型。热门语言对使用轻量级的 opus-mt 模型（约 60MB），小语种自动回退到通用的 nllb-200 模型。

💾

模型池管理

内置 LRU 模型池，智能管理多个翻译模型的生命周期。支持引用计数、自动淘汰、防重复加载，确保内存使用在可控范围内。

⚡

WebGPU 加速

自动检测 WebGPU 可用性，优先使用 GPU 加速推理。不支持时自动降级到 WASM，确保在所有现代浏览器中都能正常工作。

🔍

自动语言检测

基于 Unicode 范围的零成本启发式语言检测，覆盖 40 多种语言。支持简体中文和繁体中文的精确区分，无需额外模型加载。

🌐

DOM 自动翻译

使用 TreeWalker 遍历页面 DOM，智能合并文本节点，批量翻译后原子回写。支持 MutationObserver 监听动态内容变化。

🔧

Web Worker 支持

翻译推理可在 Web Worker 中执行，避免阻塞主线程。通过 postMessage 通信，保持 UI 流畅响应。

技术文章

为什么选择纯前端翻译？

传统的网页翻译方案通常依赖远程翻译 API，这意味着用户的文本需要发送到第三方服务器。 这不仅带来了隐私风险，还受限于网络条件和 API 配额。 翻译大师采用完全不同的方式：将翻译模型直接运行在用户的浏览器中。

通过 Transformers.js 和 ONNX Runtime 的组合， 我们可以在浏览器中运行 opus-mt 系列模型。 这些模型经过量化优化后仅需 60MB 左右的下载量， 却能提供接近商业翻译 API 的质量。

"最好的翻译方案是让用户完全控制自己的数据。纯前端翻译不仅保护隐私， 还能在没有网络连接的情况下正常工作 — 这才是真正的离线优先。"

对于需要覆盖更多语言的场景，翻译大师还支持 NLLB-200 M2M100 opus-mt 等多种模型， 通过智能路由器自动选择最优的翻译路径。
